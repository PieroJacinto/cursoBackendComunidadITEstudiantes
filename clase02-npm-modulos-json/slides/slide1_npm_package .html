📦 ¿Qué es NPM?

Node Package Manager

NPM es 3 cosas a la vez:

  • 🌐 Repositorio online: npmjs.com
  • 💻 Cliente terminal: comando npm
  • 📋 Registry: base de datos de paquetes
✅ Viene incluido con Node.js
Al instalar Node.js, NPM se instala automáticamente
// Verificar que NPM funciona: npm --version // Debería mostrar: 9.6.7 (o superior) // Ayuda básica: npm help npm help install

Dos formas de crear package.json

🎯 Método Recomendado: npm init -y

1
cd mi-api-blog npm init -y # ¡Crea package.json instantáneamente! # Usa valores por defecto inteligentes
2
# Resultado inmediato: { "name": "mi-api-blog", "version": "1.0.0", "description": "", "main": "index.js", "scripts": { "test": "echo \"Error: no test specified\" && exit 1" }, "keywords": [], "author": "", "license": "ISC" }

Editar manualmente después:
# Abrir package.json y personalizar: { "name": "mi-api-blog", "version": "1.0.0", "description": "API para blog personal", "main": "server.js", "scripts": { "start": "node server.js", "dev": "node server.js" }, "author": "Tu Nombre", "license": "MIT" }

📋 package.json - Estructura

Campos Esenciales

{ "name": "mi-api-blog", "version": "1.0.0", "description": "API para blog personal", "main": "server.js", "scripts": { "start": "node server.js", "dev": "node server.js", "test": "echo \"No tests yet\"" }, "keywords": ["blog", "api", "nodejs"], "author": "Tu Nombre ", "license": "MIT", "dependencies": {}, "devDependencies": {} }
🎯 En Clase 3 agregaremos:
"express": "^4.18.2" en dependencies

Explicación de Campos

"name": Nombre único del proyecto
"version": Versionado semántico (major.minor.patch)
"description": Qué hace tu proyecto
"main": Archivo principal (punto de entrada)
"scripts": Comandos automatizados
"dependencies": Paquetes para producción
"devDependencies": Paquetes solo para desarrollo
💡 Diferencia clave:
dependencies: Express, Sequelize (necesarios en producción)
devDependencies: Nodemon, Jest (solo desarrollo)

⚙️ Scripts NPM

Scripts Básicos

{ "scripts": { "start": "node server.js", "dev": "node server.js", "test": "echo \"No tests yet\"", "info": "node -e \"console.log('Proyecto:', require('./package.json').name)\"" } }
// Ejecutar scripts: npm start // Ejecuta "node server.js" npm run dev // Ejecuta "node server.js" npm test // Ejecuta el test (shortcut) npm run info // Ejecuta script personalizado
⚠️ Scripts especiales:
start y test NO necesitan run
Scripts personalizados SÍ necesitan npm run nombre

Scripts por Tipo de API

📝 Blog:
"scripts": { "start": "node server.js", "dev": "node server.js", "create-post": "node scripts/createPost.js" }
🍽️ Restaurante:
"scripts": { "start": "node server.js", "dev": "node server.js", "update-menu": "node scripts/updateMenu.js" }
📚 E-learning:
"scripts": { "start": "node server.js", "dev": "node server.js", "create-course": "node scripts/createCourse.js" }
🛍️ E-commerce:
"scripts": { "start": "node server.js", "dev": "node server.js", "update-stock": "node scripts/updateStock.js" }

💻 EJERCICIO: Crear tu package.json

⏱️

15 minutos - Crear package.json personalizado

1
Navegar a tu proyecto personal
cd mi-api-[tipo-proyecto] pwd # Verificar ubicación
2
Crear package.json rápidamente
npm init -y # ¡Listo! package.json creado instantáneamente
3
Personalizar package.json
// Abrir package.json en VS Code y editar: { "name": "mi-api-[tipo]", // blog, restaurante, etc "description": "API para [descripción específica]", "main": "server.js", // cambiar de index.js "scripts": { "start": "node server.js", "dev": "node server.js" }, "author": "Tu Nombre ", "license": "MIT" }
4
Crear server.js básico
// server.js console.log('🚀 Servidor iniciando...'); console.log('📦 Proyecto:', require('./package.json').name); console.log('📝 Descripción:', require('./package.json').description); console.log('✅ package.json funcionando correctamente!');
5
Probar que funciona
npm start npm run info # Debería mostrar información de tu proyecto # ¡Si funciona, ya tienes NPM configurado!
🎯 Objetivo: Verificar que npm start funciona y muestra la información de tu proyecto personal.
🆘 Si algo no funciona: Comparte tu pantalla o el error en el chat para ayudarte en tiempo real.
